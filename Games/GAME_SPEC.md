# GAME_SPEC — Единый формат описания разделов и заданий

> Этот файл — гайд для AI-агентов. Перед переоформлением или вёрсткой любого раздела — читай его первым.
> При вёрстке также обязательно читай `DESIGN_SYSTEM.md` и `CLAUDE.md` в корне проекта.

---

## 1. Терминология

| Термин | Что это |
|---|---|
| **Раздел (Section)** | Один тач-панель / экран выставки. Пример: "Креативный трек", "Разработка" |
| **Главное меню** | Стартовый экран раздела с 4 блоками |
| **Блок** | Крупная секция внутри раздела: описание, задания, видео, тест |
| **Задание (Task)** | Одна игра / активность внутри блока заданий |
| **Шаг (Step)** | Один ход / раунд внутри задания (например, 1 из 3 афиш) |
| **Профессия** | Конкретная специальность, к которой привязан контент |
| **Механика** | Тип игровой механики задания (см. раздел 4) |

---

## 2. Структура папки раздела

Каждый раздел оформляется в папке `Games/XXX/` (где XXX — номер: 001, 002, ...).

```
Games/XXX/
├── section.md              # Мета-информация раздела + главное меню
├── description.md          # Блок описания направления + профессии
├── videos.md               # Блок видео-интервью
├── test.md                 # Блок теста
└── tasks/
    ├── 01-task-slug.md     # Задание 1
    ├── 02-task-slug.md     # Задание 2
    ├── 03-task-slug.md     # Задание 3
    └── ...
```

**Правила именования:**
- Файлы заданий: `{порядковый номер}-{slug на латинице}.md` — например `01-poster.md`, `03-find-bug.md`
- Slug — короткое имя на латинице, kebab-case
- Порядок файлов = порядок заданий в меню

## 3. Формат файлов

### 3.1. `section.md` — описание раздела

```markdown
---
id: "001"
title: "Креативный трек"
slug: "creative"          # URL-сегмент: приложение будет доступно по /creative/
professions:
  - id: graphic-designer
    title: "Графический дизайнер"
  - id: ux-designer
    title: "UX-дизайнер"
blocks:
  - description
  - tasks
  - videos
  - test
---

Краткое описание раздела для внутреннего использования (не отображается в UI).
```

**Правила slug:**
- Короткое слово на английском, kebab-case
- Примеры: `creative`, `development`, `analytics`, `management`
- Используется для URL-маршрутов: `/creative/`, `/creative/tasks/`, `/creative/tasks/poster/` и т.д.

### 3.2. `description.md` — блок описания

```markdown
---
type: description
---

# Общее описание
Текст о направлении, отображается по умолчанию при входе в блок описания.

---

## Профессия: Графический дизайнер
id: graphic-designer

Текст описания профессии...

---

## Профессия: UX-дизайнер
id: ux-designer

Текст описания профессии...
```

**Навигация:** Из общего описания — тап по кнопке профессии → страница профессии.
Между профессиями — [→] / [←]. Всегда доступна кнопка [← Назад].

### 3.3. `videos.md` — блок видео

```markdown
---
type: videos
---

## Видео 1
profession: graphic-designer
title: "Графический дизайнер"
src: /videos/001/graphic-designer.mp4
subtitles: /videos/001/graphic-designer.vtt

---

## Видео 2
profession: ux-designer
title: "UX-дизайнер"
src: /videos/001/ux-designer.mp4
subtitles: /videos/001/ux-designer.vtt
```

**Навигация:** Карусель [→] / [←] между видео. Кнопка [← Назад].

### 3.4. `test.md` — блок теста (Бинго-тест)

Тест в каждом разделе — это **бинго-тест**: 8 вопросов, привязанных к конкретному эксперту (сотруднику Яндекса). Игрок отвечает на вопросы, затем видит сетку 3×3, где совпадения и расхождения с ответами эксперта подсвечиваются цветом.

```markdown
---
type: test
mechanic: bingo
expert:
  name: "Имя Фамилия"
  role: "Продакт-менеджер"        # роль эксперта (отображается в центре сетки)
  profession: product-manager     # id профессии из section.md
grid_labels:                       # 8 коротких тем для ячеек сетки 3×3 (центр = роль)
  - "напиток"
  - "локация"
  - "софт скилл"
  - "хобби"
  - "вызов"
  - "суперсила"
  - "инструмент"
  - "мотивация"
result_text: |
  Бинго! Синим цветом выделены совпадения ваших ответов.
  А красным — те, в которых твои ответы и ответы эксперта разошлись.
  Теперь ты знаешь чуть больше о внутренней жизни Яндекса и людей в нём!
---

# Intro
Это бинго! Ответь на вопросы и узнай, что объединяет тебя с {expert.role}.

# Instruction
Много ли у вас общего с нашим коллегой {expert.name}?
Выбери наиболее близкие тебе ответы и посмотри, совпал ли твой выбор с его.
В конце тебя ждёт наглядное бинго!

# Questions

## Вопрос 1
grid_label: "локация"           # какая ячейка сетки соответствует этому вопросу
prompt: "В какой локации чаще всего работаешь и почему?"
options:
  - text: "В офисе"
  - text: "Дома"
  - text: "В кафе"
  - text: "В парке"
expert_answer: "В офисе"         # ответ эксперта для сравнения

## Вопрос 2
grid_label: "напиток"
prompt: "Какой напиток чаще всего пьёшь на работе?"
options:
  - text: "Кофе"
  - text: "Чай"
  - text: "Воду"
  - text: "Сок"
expert_answer: "Кофе"

## Вопрос 3..8
...
```

**Flow экранов бинго-теста:**

```
1. INTRO — легенда
   • "Это бинго! Ответь на вопросы и узнай..."
   • [← Назад]  [Начать →]

2. QUESTIONS — экран с вопросами
   • 8 вопросов списком, у каждого 4 варианта (radio)
   • Выбор можно менять
   • Кнопка [Посмотреть результат] активна только когда все 8 отвечены
   • [← Назад]

3. RESULT — сетка бинго 3×3
   • Центральная ячейка: роль + имя эксперта
   • 8 ячеек вокруг: grid_labels
   • Совпал с экспертом → синий (--color-blue)
   • Не совпал → красно-оранжевый (--color-red)
   • Тап на ячейку → переворот, на обороте ответ эксперта
   • Справа/снизу: result_text
   • [В главное меню]
```

**Цвета сетки:**
- Совпадение: `--color-blue` (можно бледнее — `--color-bg-brand-translucent` как вариант)
- Расхождение: `--color-red` / `--color-orange`
- Цвета примерные, допускается использование приглушённых оттенков

### 3.5. Файл задания — `tasks/XX-slug.md`

Это основной и самый важный формат. Каждое задание — один файл.

```markdown
---
type: task
id: poster
title: "Афиша"
mechanic: choose
profession: graphic-designer
duration: 5
mode: group
order: 1
isLast: false
---

# Intro
Текст стартового экрана. Описывает, что предстоит сделать.

# Instruction
<!-- Опциональный блок. Если есть — показывается после "Начать",
     сворачиваемый, можно вызвать повторно иконкой в углу экрана. -->

Текст инструкции / чек-листа / подсказки.

# Steps

## Step 1
prompt: "Какая афиша подходит для открытия кафе ICE CREAM?"

### Options
- image: /assets/games/001/poster/step1-a.png
  correct: false
  explanation: "Стиль не соответствует тематике кафе"
- image: /assets/games/001/poster/step1-b.png
  correct: true
  explanation: "Яркие цвета и мороженое передают настроение"
- image: /assets/games/001/poster/step1-c.png
  correct: false
  explanation: "Слишком строгий стиль для кафе"

## Step 2
...

# Moral
Текст "морали" — почему эта тема важна, что узнал игрок.
```

## 4. Механики

Каждое задание использует ровно одну механику. Механика определяет тип взаимодействия на экране игры.

### `choose` — Выбери правильный
Игрок видит N вариантов (картинки или текст) и тапает на один.
После тапа — анимация + пояснение (правильно/неправильно).
Может быть несколько шагов (Step 1, 2, 3...).

**Формат шага:**
```
prompt: "Вопрос"
options:
  - text/image: значение
    correct: true/false
    explanation: "Пояснение"
```

### `find` — Найди на экране
Игрок видит изображение (скриншот интерфейса) и варианты ответов.
Нужно найти, что отсутствует / что не так.
После ответа — пояснение.

**Формат шага:**
```
prompt: "Что отсутствует на экране?"
image: /assets/...
options:
  - text: "Кнопка купить"
    correct: true
    explanation: "Без неё пользователь не может оформить заказ"
  - text: "Логотип"
    correct: false
    hint: "Посмотри внимательнее — логотип на месте"
```

### `sequence` — Восстанови порядок
Игрок видит перемешанные блоки и перетаскивает их в правильный порядок.
Опционально: удаление лишних блоков (корзина).

**Формат:**
```
blocks:
  - text: "Выбрать пиццу"
    order: 1
  - text: "Указать адрес"
    order: 2
  - text: "Оплатить по факсу"
    order: null          # null = лишний блок, нужно удалить
  - text: "Подтвердить заказ"
    order: 3
```

### `categorize` — Распредели по категориям
Игрок перетаскивает элементы в 2+ зоны/категории.
Элемент может подходить в несколько категорий или ни в одну.

**Формат:**
```
categories:
  - id: music
    title: "Приложение музыки"
  - id: maps
    title: "Приложение навигации"
items:
  - image/text: значение
    belongs: [music]
    explanation: "..."
  - image/text: значение
    belongs: [music, maps]
    explanation: "Подходит для обоих"
  - image/text: значение
    belongs: []
    explanation: "Не подходит ни к одному"
```

### `match` — Соотнеси пары
Игрок соединяет элементы 1:1 (например, код ↔ картинка, аватар ↔ язык).

**Формат:**
```
pairs:
  - left:
      type: image/text
      value: "Аватар джентльмена"
    right:
      type: image/text
      value: "Java"
    explanation: "Java требует строгой типизации..."
  - left: ...
    right: ...
```

### `label` — Разметка
Игрок помечает каждый элемент одной из N меток (например: "безопасно" / "опасно").

**Формат:**
```
labels:
  - id: safe
    title: "Безопасно"
    icon: "checkmark"
  - id: danger
    title: "Опасно"
    icon: "warning"
items:
  - content: "Описание или изображение элемента"
    correctLabel: danger
    explanation: "Поддельный домен отправителя..."
```

### `mark` — Отметь на макете
Игрок тапает по точкам на изображении-макете, отмечая проблемные места.
По завершении — кнопка "Проверить".

**Формат:**
```
image: /assets/...
hints: "Текст отзывов / подсказок рядом с макетом"
targets:
  - area: { x: 120, y: 340, radius: 40 }
    explanation: "Пользователи жаловались на сложность поиска профиля"
  - area: { x: 500, y: 200, radius: 30 }
    explanation: "Кнопка регистрации не работает"
```

### `catch` — Лови объекты (аркада)
Игрок управляет "ловушкой" (сачок, корзинка) внизу экрана. Сверху падают объекты (жуки, элементы...).
Нужно поймать все / нужные. После поимки каждого — показывается карточка с описанием.

**Формат:**
```
catcher:
  type: net              # net | basket | hand
  label: "Сачок"
objects:
  - icon: /assets/games/XXX/catch/bug-1.png
    title: "Кнопка не нажимается"
    description: "UI-баг: элемент не реагирует на клик из-за перекрывающего слоя"
    category: ui
  - icon: /assets/games/XXX/catch/bug-2.png
    title: "Утечка данных"
    description: "Баг безопасности: персональные данные передаются без шифрования"
    category: security
  - icon: /assets/games/XXX/catch/bug-3.png
    title: "Бесконечная загрузка"
    description: "Баг производительности: запрос к серверу зацикливается"
    category: performance
```

### `bingo` — Бинго-тест
Игрок отвечает на 8 вопросов (у каждого 4 варианта). Ответы сравниваются с ответами реального эксперта (сотрудника Яндекса). Результат — сетка 3×3: в центре роль эксперта, 8 ячеек = темы вопросов. Совпадения подсвечиваются синим, расхождения — красным/оранжевым. Каждая ячейка «переворачивается» по тапу, показывая ответ эксперта.

Используется как основная механика тестов (блок `test.md`) во всех разделах.

**Формат:** см. раздел 3.4 (`test.md`).

**Ключевые отличия от обычного quiz:**
- Нет «правильных» ответов — только совпадение/расхождение с экспертом
- Все 8 вопросов на одном экране (список), не последовательно
- Результат визуализируется как сетка бинго, а не список ✅/❌
- Ячейки интерактивные (переворот по тапу)

### `quiz` — Тест / викторина
Последовательные вопросы с вариантами. Может использоваться в заданиях.
Формат аналогичен `choose`, но без изображений, только текст.

## 5. Единый flow экранов

Все задания проходят по одной последовательности экранов:

```
[Главное меню раздела]
  ├── Блок описания
  │     ├── Общее описание направления
  │     └── Профессия 1 ↔ Профессия 2 ↔ ...
  │
  ├── Блок заданий (список карточек)
  │     └── Задание
  │           ├── 1. INTRO — стартовый экран
  │           │     • Текст описания задачи
  │           │     • Метаданные: ⏱ время, 👤/👥 тип
  │           │     • [← Назад]  [Начать →]
  │           │
  │           ├── 2. INSTRUCTION (опционально)
  │           │     • Чек-лист / правила / подсказка
  │           │     • Закрывается на ✕, вызывается повторно иконкой
  │           │
  │           ├── 3. GAME — экран игры
  │           │     • Шаги (Step 1..N) по механике
  │           │     • [← Назад] всегда доступен
  │           │     • Обратная связь: после каждого шага ИЛИ в конце
  │           │
  │           ├── 4. RESULT — экран результата
  │           │     • ✅ Правильные + пояснения
  │           │     • ❌ Ошибки + пояснения
  │           │
  │           ├── 5. MORAL — текстовый блок
  │           │     • Почему это важно / что узнал игрок
  │           │
  │           └── 6. NAVIGATION
  │                 • [→ Следующее задание] — если не последнее
  │                 • [← В меню] — если последнее (isLast: true)
  │
  ├── Блок видео — карусель по профессиям
  │     • Воспроизведение с субтитрами
  │     • [←] [→] между видео
  │
  └── Блок тест — викторина
```

## 6. Спецэлементы внутри контента

### Кликабельные термины (тултипы)
Внутри текстов заданий некоторые слова нужно сделать кликабельными — по тапу всплывает пояснение.

**Формат в тексте:**
```
Мой код состоит из [классов]{tooltip: "Класс — это шаблон для создания объектов"}
и строгих [определений типов]{tooltip: "Тип определяет, какие данные может хранить переменная"}.
```

### Контент в шагах
Шаги могут содержать:
- `text:` — текстовый контент
- `image:` — путь к изображению
- `code:` — блок кода (отображается моноширинным шрифтом)
- `hidden: true` — контент скрыт плашкой, открывается по кнопке "Подсказка"

## 7. Обратная связь (Feedback)

Два режима обратной связи в заданиях:

### Мгновенный (`feedback: instant`)
После каждого действия игрока сразу показывается результат:
- ✅ Зелёная подсветка + пояснение
- ❌ Красная подсветка + подсказка / пояснение
- Анимация (переворот, отталкивание, подсветка...)

### По завершении (`feedback: onComplete`)
Игрок выполняет все действия, потом нажимает [Проверить].
Показывается сводка: все правильные и неправильные ответы с пояснениями.

**Указывается в frontmatter задания:**
```yaml
feedback: instant    # или onComplete
```

## 8. Чек-лист для агента: переоформление сырого ТЗ

Когда получаешь папку с сырым ТЗ (набор .md файлов от сценаристов):

1. **Прочитай все файлы** в папке
2. **Определи раздел:** найди файл "Основа" или аналог — извлеки название, список профессий, структуру блоков
3. **Создай `section.md`** с метаданными
4. **Создай `description.md`** — собери общее описание + описания профессий
5. **Создай `videos.md`** — перечисли видео по профессиям
6. **Создай `test.md`** — если есть тест, оформи вопросы
7. **Для каждого задания:**
   - Определи механику из списка (раздел 4)
   - Определи mode (group/solo), duration, profession
   - Извлеки Intro, Instruction (если есть), Steps, Moral
   - Разложи контент шагов: prompt, options/blocks/pairs/items
   - Определи feedback mode: instant или onComplete
   - Запиши в `tasks/XX-slug.md`
8. **Проверь:**
   - У каждого задания есть `order` и `isLast`
   - У каждого шага есть `explanation` для каждого варианта
   - Все ассеты указаны путями `/assets/games/XXX/...`
   - Тултипы оформлены как `[слово]{tooltip: "пояснение"}`

## 9. Чек-лист для агента: вёрстка раздела

Когда получаешь переоформленную папку и нужно сверстать:

1. **Прочитай `DESIGN_SYSTEM.md`** и **`CLAUDE.md`** — это закон
2. **Прочитай `section.md`** — пойми структуру раздела
3. **Создай страницу раздела** `src/pages/SectionXXX/SectionXXX.tsx`
4. **Главное меню** — используй компоненты `Card`, `Button`, `Container`
5. **Блок описания** — `Container` + текст + кнопки профессий (`Button type="secondary"`)
6. **Блок заданий** — список `Card` с метаданными (иконка 👤/👥, время)
7. **Каждое задание** — отдельный компонент или экран с flow:
   - Intro → Instruction → Game → Result → Moral
8. **Механики** — реализуй согласно типу (drag-drop, тап, разметка...)
9. **Навигация** — `IconButton` для [← Назад], `Button` для [Начать], [→ Далее]
10. **Обратная связь** — `PopUp` для пояснений, цвета `--color-green` / `--color-red`
11. **Мораль** — `PopUp` или отдельный экран с текстом
12. **Используй только токены** — никаких хардкод-значений

## 10. Маппинг механик → UI-компоненты

| Механика | Основные компоненты | Взаимодействие |
|---|---|---|
