# Intro
Пользователи и заказчики постоянно просят добавить в приложение новые фичи. Но если сделать всё — приложение станет тяжёлым монстром, а команда выгорит. Твоя задача — защитить бэклог от мусорных идей, убрав лишние.

# Instruction
По экрану сверху вниз одна за другой падают карточки с идеями для приложения «Доставка еды».

- Смахни в корзину (свайп влево/вправо) — если идея мусорная
- Пропусти вниз (не трогай) — если идея полезная

Карточки падают последовательно — следующая появляется после решения по текущей. Прогресс отображается вверху экрана.

# Steps

## Step 1
prompt: "Отфильтруй мусорные идеи из бэклога"

### Objects
- text: "Добавить криптокошелек в доставку пиццы"
  shouldRemove: true
  explanation: "Криптокошелёк не имеет отношения к доставке еды. Это мусорная идея."

- text: "Бесконтактная оплата"
  shouldRemove: false
  explanation: "Полезная функция — упрощает процесс оплаты для пользователя."

- text: "Отслеживание курьера на карте"
  shouldRemove: false
  explanation: "Классическая и нужная функция — пользователь знает, когда ждать заказ."

- text: "Сделать кнопку заказа невидимой"
  shouldRemove: true
  explanation: "Невидимая кнопка — это UX-катастрофа. Никто не сможет сделать заказ."

- text: "Встроить мини-игру «Тамагочи-курьер»"
  shouldRemove: true
  explanation: "Развлечение, не связанное с основной ценностью продукта. Отвлекает команду."

- text: "Фильтр по аллергенам"
  shouldRemove: false
  explanation: "Важная функция для здоровья пользователей. Реальная потребность."

- text: "Добавить гороскоп дня к каждому заказу"
  shouldRemove: true
  explanation: "Не имеет отношения к доставке. Мусорная идея."

- text: "Оценка и отзывы о ресторанах"
  shouldRemove: false
  explanation: "Помогает пользователям делать осознанный выбор. Стандарт индустрии."

- text: "Интеграция с соцсетями для публикации каждого заказа"
  shouldRemove: true
  explanation: "Навязчивая функция. Мало кто хочет публиковать свои заказы еды."

- text: "Повтор предыдущего заказа одной кнопкой"
  shouldRemove: false
  explanation: "Удобная функция — экономит время для постоянных клиентов."

# Moral
Хороший продукт — это не тот, в котором есть всё. Это тот, в котором есть нужное. Главный навык продакта — уметь говорить «нет».
