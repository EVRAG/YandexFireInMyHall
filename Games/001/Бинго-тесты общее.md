# Бинго-тест — типовой сценарий

> Механика `bingo`. Используется как блок "Тест" в каждом разделе (тач-панели).
> Каждый тач содержит один бинго-тест, привязанный к одной специальности и конкретному эксперту.

---

## Концепция

Бинго-тест — это не проверка знаний, а **сравнение предпочтений** игрока с ответами реального сотрудника Яндекса. Нет правильных/неправильных ответов — есть совпадения и расхождения.

Результат визуализируется как сетка бинго 3×3, где каждая ячейка представляет тему одного из 8 вопросов.

---

## Flow экранов

### 1. Экран легенды (Intro)

- Текст: *«Это бинго! Ответь на вопросы и узнай, что объединяет тебя с {роль эксперта}»*
- Кнопки: [← Назад] [Начать →]

### 2. Экран вопросов (Questions)

- Вступительный текст: *«Много ли у вас общего с нашим коллегой {имя}? Выбери наиболее близкие тебе ответы и посмотри, совпал ли твой выбор с его. В конце тебя ждёт наглядное бинго!»*
- На экране **списком** представлены **8 вопросов**
- У каждого вопроса **4 варианта ответа** (radio — один выбор)
- Выбор можно менять до отправки
- Кнопка **[Посмотреть результат]** — становится активна, когда выбраны ответы на все 8 вопросов
- Кнопка [← Назад]

### 3. Экран результата (Result) — сетка бинго

**Сетка 3×3:**

```
┌──────────┬──────────┬──────────┐
│  тема 1  │  тема 2  │  тема 3  │
├──────────┼──────────┼──────────┤
│  тема 4  │  РОЛЬ /  │  тема 5  │
│          │   ИМЯ    │          │
├──────────┼──────────┼──────────┤
│  тема 6  │  тема 7  │  тема 8  │
└──────────┴──────────┴──────────┘
```

- **Центральная ячейка:** название роли (+ имя эксперта, если помещается)
- **8 ячеек вокруг:** краткие темы вопросов (`grid_labels`)

**Цвета ячеек:**
- Совпал с экспертом → **синий** (приглушённый оттенок)
- Не совпал → **красно-оранжевый** (приглушённый оттенок)
- Цвета примерные, допускается использование бледных вариантов

**Интерактив:**
- Тап на ячейку → **анимация переворота** (flip)
- На обратной стороне — ответ эксперта на этот вопрос

**Итоговый текст** (справа или снизу от сетки, в зависимости от ориентации):

> Бинго! Синим цветом выделены совпадения ваших ответов.
> А красным — те, в которых твои ответы и ответы эксперта разошлись.
> Теперь ты знаешь чуть больше о внутренней жизни Яндекса и людей в нём!
> Впереди нас ждёт ещё много всего интересного: знакомство с другими ИТ-профессиями и новые знания.
> Увидимся на следующем стенде!

- Кнопка **[В главное меню]**

## Требования к данным от клиента

Для каждого бинго-теста (для каждого раздела) клиент предоставляет:

1. **Эксперт:** имя, роль, профессия
2. **8 вопросов** с 4 вариантами ответа каждый
3. **Ответы эксперта** на каждый вопрос
4. **8 кратких тем** (grid_labels) для отображения в ячейках сетки
5. **Итоговый текст** (или используется дефолтный)

---

## Примеры тем вопросов (из концепта)

| Тема (grid_label) | Примерный вопрос |
|---|---|
| напиток | Какой напиток чаще всего пьёшь на работе? |
| локация | В какой локации чаще всего работаешь? |
| софт скилл | Какой софт скилл считаешь самым важным? |
| хобби | Чем занимаешься в свободное время? |
| вызов | Какой главный вызов в работе? |
| суперсила | Какая у тебя суперсила? |
| инструмент | Какой рабочий инструмент самый важный? |
| мотивация | Что мотивирует тебя в работе? |

---

## Маппинг на UI-компоненты

| Элемент | Компонент |
|---|---|
| Экран вопросов | Список `Card` с radio-вариантами внутри |
| Варианты ответа | `Button type="secondary"` или radio-группа |
| Сетка бинго | Кастомная CSS Grid 3×3 с flip-анимацией |
| Ячейка (лицо) | Тема вопроса + цвет совпадения |
| Ячейка (оборот) | Ответ эксперта |
| Итоговый текст | Текстовый блок справа/снизу от сетки |
| Кнопка результата | `Button type="main"` — «Посмотреть результат» |
| Кнопка выхода | `Button type="main"` — «В главное меню» |
